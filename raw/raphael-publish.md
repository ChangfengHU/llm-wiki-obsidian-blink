### 🎯 一句话定位
一个开箱即用的**纯前端 Markdown 转微信公众号富文本排版工具**，通过底层 DOM 重塑和图片转码，彻底解决微信编辑器“图片防盗链”和“样式塌陷”的痛点。

### ⚡ 核心功能
1. **微信专属兼容引擎 (`wechatCompat`)**：一键复制时，自动将外链图片转为 Base64（绕过微信防盗链报错），并重写列表/表格的 DOM 结构以防止微信编辑器吞样式。
2. **富文本“魔法粘贴”净化**：内置 `turndown`，支持从 Notion、飞书、Word 直接复制富文本并瞬间转化为纯净 Markdown，支持直接粘贴图片。
3. **30套开箱即用的高定主题**：深度定制的 CSS 样式（涵盖代码高亮、引用、表格等），不仅限于改个主色调，而是全局排版风格的切换。
4. **多端预览与导出**：支持 PC/平板/手机实时响应式预览，并集成 `html2pdf.js` 支持直接导出 PDF 或 HTML。

### 🎭 适用场景
- **技术博主发文**：在本地或 GitHub/Notion 用 Markdown 写好包含大量代码块的技术文章，需要无损同步到微信公众号。
- **新媒体/公众号运营**：苦于市面上排版工具太臃肿或收费，需要一个轻量、极速、免登录的网页端排版工具。
- **文档沉淀与分发**：将日常 Markdown 笔记快速渲染为精美的 PDF 文件用于存档或邮件分享。

### ✅ 是否值得深入研究
**结论：值得（针对特定领域）。**
- **作为工具**：极具实用价值，直接加入书签日常使用即可。
- **作为源码**：非常适合**前端工程师**学习。它的技术栈极度现代化（React 18 + Vite + TS + Tailwind），且具备完整的单元测试和 E2E 测试。核心价值在于 `src/lib/wechatCompat.ts`（微信 DOM 兼容黑魔法）和 `htmlToMarkdown/turndown` 的定制实现，是学习富文本/Markdown AST 转换的优秀范例。

### ⚠️ 主要缺点或风险
1. **Base64 图片体积风险**：将所有图片转为 Base64 复制到剪贴板，如果文章包含大量高清大图，可能会导致剪贴板内存溢出，或在粘贴到微信时卡顿/触发微信单篇文章体积限制。
2. **过度依赖微信编辑器规则**：微信公众平台一旦修改其底层富文本过滤规则（Sanitizer），该项目的 `wechatCompat` 引擎可能随时失效，需要持续跟进维护。
3. **缺乏云端能力**：纯前端静态项目，不支持文章云端保存和历史版本管理，刷新或清缓存可能导致未导出的草稿丢失。

### 🔗 与同类项目对比
- **对比 Markdown Nice (mdnice)**：mdnice 是目前国内最火的同类工具，但已高度商业化（各种扫码登录、VIP 主题）。Raphael Publish 优势在于**完全开源、纯净无广、无需登录**，且“飞书/Notion 魔法粘贴”体验更好；劣势在于缺少 mdnice 庞大的用户 UGC 主题社区。
- **对比 Doocs/md**：两者定位类似，但 Raphael Publish 的 UI 设计更具现代感，且在“微信防盗链突破（自动 Base64）”和“响应式预览”的工程实现上更加激进和彻底。
