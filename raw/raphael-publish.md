### 🎯 一句话定位
一款专为微信公众号打造的现代 Markdown 渲染引擎，通过“富文本逆向转 MD”和“底层 DOM 兼容处理”，解决跨平台内容同步与微信排版的痛点。

### ⚡ 核心功能
1. **魔法粘贴（富文本净化）**：基于 `turndown`，支持从飞书、Notion、Word 直接复制富文本，粘贴瞬间自动转换为纯净 Markdown，支持直接粘贴截图。
2. **微信级深度兼容**：内置 `wechatCompat` 引擎重塑 DOM（防止列表/表格塌陷），并在复制时自动将外链图片转为 Base64，彻底解决微信“此图片来自第三方”的防盗链报错。
3. **30+ 现代预设主题**：告别传统简陋样式，提供 Notion、Claude、纽约时报等高品质定制排版，一键切换。
4. **多端预览与多格式导出**：支持手机/平板/PC实时响应式预览，并可通过 `html2pdf.js` 导出为 PDF 或 HTML。

### 🎭 适用场景
- **公众号内容分发**：技术博主、自媒体人将写好的 Markdown 文章一键美化并无缝粘贴到微信公众号后台。
- **跨平台创作者**：习惯在 Notion、飞书、Obsidian 中写作，需要将内容带格式搬运到其他富文本平台的作者。
- **快速文档美化**：需要将一段干瘪的 Markdown 快速渲染成高颜值 PDF（如简历、报告）进行分享的用户。

### ✅ 是否值得深入研究
**结论：值得（特别是前端工程师与工具开发者）**

**理由**：
1. **技术栈极度现代且标准**：React 18 + TS + Vite + Tailwind，配合 Vitest 和 Playwright 测试，是一个非常优秀的现代前端工程化模板。
2. **极具参考价值的业务逻辑**：源码中的 `htmlToMarkdown.ts`（富文本逆向解析）和 `wechatCompat.ts`（针对特定富文本编辑器的 DOM 劫持与兼容）是解决实际业务痛点的绝佳代码范例。

### ⚠️ 主要缺点或风险
1. **微信规则变动风险**：过度依赖微信后台当前的编辑器逻辑，如果微信更新编辑器底层机制，或者对大体积 Base64 图片进行限制，核心功能（一键复制）可能失效。
2. **数据持久化弱**：作为一个纯前端工具，README 未提及云端同步或强大的本地缓存机制，作为主力“编辑器”使用时存在数据丢失风险，更适合作为“排版中转站”。
3. **大文档性能未知**：React 状态驱动的实时 Markdown 渲染，在处理超长图文或海量 Base64 图片时，可能存在卡顿风险。

### 🔗 与同类项目对比
*对比目标：Markdown Nice (mdnice), doocs/md*
- **优势**：**颜值与体验降维打击**。相比 mdnice 传统的 UI 和古板的主题，该项目设计极其现代（类似 Vercel/Notion 风格）；“魔法粘贴”功能对非 Markdown 原生用户极其友好。
- **劣势**：**生态与可扩展性不足**。mdnice 拥有庞大的用户自定义主题库和活跃的社区生态，而本项目目前主要依赖内置的 30 套主题，自定义扩展能力和插件生态尚不如老牌工具成熟。
