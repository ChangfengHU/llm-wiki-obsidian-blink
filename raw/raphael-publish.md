# 📦 项目：raphael-publish
**项目主页**: [https://github.com/liuxiaopai-ai/raphael-publish](https://github.com/liuxiaopai-ai/raphael-publish)
**核心描述**: Raphael Publish - 公众号排版大师 | 现代 Markdown 排版引擎
**技术标签**: markdown, markdown-editor, wechat

### 🎯 一句话定位
一款专为微信公众号打造的现代前端 Markdown 排版引擎，核心解决富文本一键净化、外链图片防盗链突破及微信特殊 DOM 兼容问题。

### ⚡ 核心功能（3~5条）
1. **富文本“魔法”粘贴**：底层集成 `turndown`，支持将 Notion、飞书、Word 等复杂富文本直接粘贴并精准降级为纯净 Markdown，免去手动重写语法的繁琐。
2. **微信底层 DOM 适配**：通过核心的 `wechatCompat` 引擎，重塑列表、表格及多图网格布局的 DOM 结构，彻底解决微信编辑器中常见的样式塌陷与折断问题。
3. **图片防盗链绕过**：一键复制到公众号时，自动将外链图片转换为 Base64 编码，避免出现“此图片来自第三方”的报错。
4. **开箱即用的高定主题**：内置 30 套深度定制的 CSS 样式（如 Notion、Medium、Bloomberg 风格），支持多端（手机/平板/PC）实时响应式预览。

### 🎭 适用场景
- **跨平台内容创作者**：习惯在 Notion、Obsidian 或飞书中进行结构化写作，需要无缝、无损地将内容同步发布到微信公众号。
- **前端工程师学习参考**：需要研究或开发富文本编辑器、Markdown 解析器，特别是需要解决微信富文本底层兼容“黑魔法”的开发者。
- **企业内部工具搭建**：团队需要定制一个轻量级、现代化的多渠道图文排版与分发系统，可将其作为底层引擎进行二次开发。

### ✅ 是否值得深入研究
**值得（特别是对于前端工程师）**。
项目代码库非常现代化且干净（React 18 + Vite + Tailwind + TypeScript），工程化规范良好（包含 Vitest 单元测试和 Playwright E2E 测试）。其核心价值在于 `src/lib/wechatCompat.ts` 和 `htmlToMarkdown.ts` 两个文件，提供了处理复杂 DOM 转换和微信生态兼容性问题的优秀实战范例。

### ⚠️ 主要缺点或风险
1. **Base64 图片体积限制**：将所有外链图片转为 Base64 虽巧妙绕过了防盗链，但如果文章包含大量高清大图，会导致单篇内容体积暴增，可能触发微信后台的保存限制或影响最终用户的加载性能。
2. **缺乏数据持久化机制**：作为一个纯前端工具，目前侧重于“排版与中转”，缺乏云端同步或完备的本地草稿箱管理，作为主力编辑器存在数据丢失风险。
3. **平台规则依赖**：高度依赖微信公众号当前编辑器的 DOM 解析规则，若微信底层逻辑更新，当前的兼容 hack 代码可能失效，需要持续维护。

### 🔗 与同类项目对比
- **对比 Markdown Nice (mdnice)**：mdnice 历史悠久但目前商业化较重，代码有一定历史包袱。Raphael 技术栈极新，UI 交互更现代（引入了 framer-motion），且完全免费开源。
- **对比 Doocs/md**：Doocs 也是优秀的开源排版工具，但 Raphael 在“逆向解析”（富文本转 MD 的魔法粘贴功能）体验上打磨得更细致，且默认提供的 30 套主题更符合当下科技圈与设计圈的现代审美。
