### 🎯 一句话定位
一款专为微信公众号打造的现代 Markdown 排版引擎，通过富文本自动转换和 Base64 图片内联，彻底解决公众号排版繁琐与外部图片防盗链的痛点。

### ⚡ 核心功能
*   **富文本“魔法粘贴”**：基于 `turndown`，支持从飞书、Notion、Word 直接复制内容并自动逆向解析为纯净 Markdown，支持直接粘贴截图。
*   **微信生态深度适配（杀手锏）**：一键复制到公众号时，底层重塑 DOM 解决列表/表格塌陷问题；**外链图片自动转 Base64**，直接绕过微信“此图片来自第三方”的防盗链报错。
*   **开箱即用的高定主题**：内置 30 套审美在线的主题（如 Notion风、Claude风、少数派风），代码高亮、背景色、间距等细节已调优。
*   **多端实时预览与导出**：支持手机/平板/PC 视图切换，支持导出 PDF 和 HTML 格式。

### 🎭 适用场景
1.  **公众号运营者/自媒体**：习惯在 Notion、飞书、Obsidian 中写作，需要快速、无损地将内容排版并发布到微信公众号。
2.  **技术博主/程序员**：需要发布包含复杂代码块、表格的技术文章，厌倦了传统排版工具的繁琐微调。
3.  **前端工程师学习**：想要研究如何基于 `markdown-it` 和 React 处理复杂的 AST/DOM 节点转换，以及如何实现优雅的纯前端工具。

### ✅ 是否值得深入研究
**值得（针对前端开发与内容创作者）。**
*   **作为工具**：体验极佳，UI 现代，切中痛点，可以直接替代现有的排版工具。
*   **作为源码**：工程化规范极好。采用 React 18 + Vite + TS 现代技术栈，目录结构清晰。更重要的是，项目包含了完整的单元测试（Vitest）和端到端测试（Playwright），这在个人开源工具中非常少见，是学习富文本处理、前端测试驱动开发（TDD）的优秀范例。

### ⚠️ 主要缺点或风险
1.  **Base64 图片的隐性限制**：将图片转为 Base64 虽然解决了防盗链，但如果文章包含大量高清大图，会导致单次复制的 HTML 体积过大，可能触发微信公众号编辑器底层的字符数或体积限制导致保存失败。
2.  **纯工具属性，缺乏云端能力**：目前是一个纯前端无状态应用，没有图床管理、历史记录云端同步等功能，刷新浏览器可能导致未保存数据丢失。
3.  **受制于微信规则**：高度依赖微信编辑器现有的 DOM 解析规则，若微信后台进行大改版，`wechatCompat` 兼容引擎可能需要频繁跟进维护。

### 🔗 与同类项目对比
*   **对比 Markdown Nice (mdnice)**：mdnice 是老牌霸主，但目前商业化严重、广告多、UI 偏传统。Raphael Publish 完全开源纯净，基于 Tailwind CSS 构建，UI 更现代，主题审美更贴合当下（如 Claude、Notion 风格）。
*   **对比 Doocs/md**：Doocs 也是优秀的开源竞品。Raphael 的优势在于其“魔法粘贴”功能（跨平台富文本逆向解析能力更强）和前端工程化水平（代码质量更高，测试覆盖更全）。
