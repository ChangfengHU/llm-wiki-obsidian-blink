### 🎯 一句话定位
一款专为微信公众号打造的纯前端 Markdown 排版引擎，核心解决富文本粘贴净化、微信样式兼容（防塌陷/图片防盗链）以及多主题排版痛点。

### ⚡ 核心功能（3~5条）
- **魔法粘贴（富文本逆向解析）**：复制飞书、Notion 或网页富文本，粘贴时自动通过 `turndown` 转化为纯净 Markdown，支持直接粘贴图片。
- **微信深度兼容引擎**：底层重塑 DOM（`wechatCompat.ts`），解决微信公众号中列表、表格塌陷问题；外链图片自动转 Base64，彻底绕过“图片来自第三方”的防盗链报错。
- **30套开箱即用的高定主题**：内置 Notion、GitHub、少数派等多种高质量 CSS 样式，覆盖字体、代码块、引用等细节，一键切换。
- **多图网格排版**：支持多图并排布局，且确保在微信公众号极端的渲染环境下不折断、不乱版。
- **多端预览与导出**：支持手机/平板/PC实时预览，并可直接导出为 PDF 或 HTML。

### 🎭 适用场景
- **公众号内容运营**：平时在 Notion/飞书/Word 写稿，需快速排版发布到微信公众号，拒绝手动调格式。
- **技术博主/独立开发者**：需要将包含大量代码块、表格的 Markdown 文章，无损且美观地发布到微信生态。
- **前端工程师学习**：需要研究富文本与 Markdown 相互转换、剪贴板劫持处理、以及微信公众号 DOM 兼容黑魔法。

### ✅ 是否值得深入研究
**结论：值得（特别是对前端工程师）。**
**理由**：项目技术栈现代（React 18 + Vite + Tailwind），代码结构清晰。其核心价值不在于 UI，而在于 `src/lib/wechatCompat.ts` 和 `htmlToMarkdown.ts`。如果你正在做富文本编辑器、内容发布平台，或者需要处理恶心的微信兼容问题，这个项目的源码能为你节省大量踩坑时间。

### ⚠️ 主要缺点或风险
- **微信规则变动风险**：重度依赖微信现有的黑盒渲染规则。若微信公众号后台升级编辑器或限制 Base64 数据大小，其兼容引擎可能瞬间失效。
- **纯前端架构的性能瓶颈**：通过将所有外链图片转为 Base64 来解决防盗链，如果文章包含大量高清大图，会导致剪贴板数据极其庞大，粘贴时可能引发浏览器卡顿或微信后台崩溃。
- **缺乏图床与云端能力**：作为纯静态工具，无法管理历史文章，也不支持自动上传图片到自有图床，长远使用存在局限。

### 🔗 与同类项目对比
- **对比 Markdown Nice (MdNice)**：
  - **优势**：UI 更现代，交互更顺滑。主打的“魔法粘贴”体验更好，降低了非技术人员的使用门槛（不需要精通 MD 语法）；Base64 图片转换方案比配置图床更小白。
  - **劣势**：生态和社区远不如 MdNice。MdNice 拥有庞大的第三方主题库和插件生态，且支持多平台分发（知乎、掘金等），而本项目目前强绑定微信公众号。
- **对比 wechat-format 等老牌排版器**：技术栈更先进，代码可维护性极高，去除了老旧的 jQuery/Vue2 历史包袱。
