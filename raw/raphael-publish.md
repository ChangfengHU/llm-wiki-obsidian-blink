### 🎯 一句话定位
一款专为微信公众号打造的纯前端 Markdown 排版引擎，核心解决跨平台富文本粘贴与微信防盗链、样式塌陷等兼容性痛点。

### ⚡ 核心功能
1. **微信生态深度兼容 (`wechatCompat`)**：一键复制到公众号时，自动将外链图片转为 Base64（彻底绕过微信“此图片来自第三方”防盗链），并重塑底层 DOM 解决列表/表格在微信中的塌陷问题。
2. **魔法粘贴与净化 (`htmlToMarkdown`)**：劫持剪贴板，支持从飞书、Notion、Word 复制富文本，利用 `turndown` 自动逆向解析为纯净的 Markdown 语法，支持直接粘贴图片。
3. **30套高定主题系统**：内置经过精细 CSS 调优的主题库（涵盖 Notion、少数派、GitHub 等风格），一键切换，覆盖背景、代码高亮、引用等细节。
4. **多端预览与导出**：支持移动端/平板/PC 实时响应式预览，并集成 `html2pdf.js` 实现高质量 PDF 导出。

### 🎭 适用场景
- **跨平台创作者**：习惯在 Notion、飞书或 Obsidian 中写作，需要无损、免排版地将内容同步至微信公众号。
- **技术博主**：文章包含大量代码块、表格，需要确保在公众号发布时代码高亮美观且不乱码。
- **轻量级排版与分享**：需要快速将一段 Markdown 文本渲染成高颜值文档，并导出为 PDF 或 HTML 存档。

### ✅ 是否值得深入研究
**结论：值得（特别是前端工程师）。**
**理由**：
1. **工程化极佳**：基于 Vite + React 18 + TS 构建，代码结构清晰，且配置了完整的 E2E 测试（Playwright）和单元测试（Vitest），是现代前端开源项目的优秀模板。
2. **核心逻辑有极高参考价值**：对于想学习**剪贴板事件劫持 (Clipboard API)**、**AST/富文本与 Markdown 互转**、以及**处理微信等封闭生态 DOM 兼容性**的开发者来说，源码中的 `src/lib/` 目录（特别是 `wechatCompat.ts` 和 `htmlToMarkdown.ts`）是极佳的学习素材。

### ⚠️ 主要缺点或风险
1. **Base64 转换的隐患**：将图片转为 Base64 虽解决了防盗链，但微信公众号后台对单篇文章的总字符数有隐性限制。如果文章包含大量高清大图，全部转为 Base64 可能导致保存失败或卡顿。
2. **纯前端架构的局限**：作为一个纯静态 SPA，没有后端数据库。如果用户未及时导出或复制，清理浏览器缓存可能导致草稿丢失（依赖本地存储）。
3. **排版上限受限于 Markdown**：不支持复杂的富文本拖拽布局（如分栏、自由浮动图片），仅适合流式文本排版。

### 🔗 与同类项目对比
- **相比 Markdown Nice (mdnice)**：mdnice 历史悠久但代码库较老，且逐渐商业化（需登录、部分主题收费）。本项目完全开源免费，技术栈更现代（React 18 + Tailwind），UI 审美更偏向现代极简风。
- **相比 Doocs/md (微信 Markdown 编辑器)**：功能定位相似，但本项目的“魔法粘贴”（富文本逆向解析）体验更佳，且对多图网格排版和微信特定 DOM 的处理更为精细，开箱即用的主题质量更高。
