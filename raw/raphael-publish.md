### 🎯 一句话定位
一款基于纯前端技术栈的 Markdown 渲染与排版引擎，专为解决微信公众号富文本复制格式错乱、图片防盗链等痛点而生。

### ⚡ 核心功能
1. **魔法粘贴**：利用 `turndown` 自动将飞书、Notion 等富文本及剪贴板图片转化为纯净 Markdown，免除手写语法。
2. **微信底层适配**：点击复制时，自动将外链图片转 Base64 破解防盗链，并通过 `wechatCompat` 重塑 DOM 防止表格和列表在微信中塌陷。
3. **海量高定主题**：内置 30 套精美的纯 CSS 主题，一键切换即可改变全局排版风格。
4. **多端预览与导出**：支持手机、平板、PC 实时响应式预览，并可直接导出为 PDF 或 HTML 文件。

### 🎭 适用场景
1. **公众号日常推文**：习惯使用 Markdown 写作的号主，需要快速套用精美样式并发布到微信后台。
2. **跨平台内容分发**：创作者需要将飞书、Notion、知乎等平台的图文内容，无损搬运至微信公众号。
3. **前端工具链学习**：开发者需要一个结构清晰、包含完整测试用例的纯前端富文本处理与剪贴板劫持的参考实现。

### ✅ 是否值得深入研究
**值得（针对特定领域）。**
如果你正在开发富文本编辑器、处理复杂的剪贴板数据（Clipboard API），或者需要破解微信公众号的排版限制，它的 `src/lib` 目录（特别是 `wechatCompat.ts` 和 `htmlToMarkdown.ts`）极具源码阅读价值。项目工程化标准高（自带 Vitest/Playwright），是优秀的纯前端微型工具范例；但对于资深架构师而言业务逻辑偏简单，扫一眼核心库即可。

### ⚠️ 主要缺点或风险
1. **Base64 图片体积限制**：将图片转 Base64 虽能绕过防盗链，但多图或大图长文会导致单次复制的 HTML 字符串体积过大，可能触发微信后台限制或造成浏览器卡顿。
2. **纯本地存储风险**：作为纯前端无后端的工具，缺乏云端同步与持久化存储机制，清理浏览器缓存会导致未保存的内容丢失。
3. **高度依赖开源生态**：核心解析逻辑强依赖 `markdown-it` 和 `turndown`，若需深度定制非标准 Markdown 语法，扩展成本较高。

### 🔗 与同类项目对比
与最著名的老牌竞品 **Markdown Nice (mdnice)** 相比：
**差异化优势**：技术栈更现代（React 18 + Vite），UI 更极简；自带外链图片转 Base64 功能，**直接免去了新手配置图床（PicGo）的繁琐步骤**，开箱即用体验极佳。
**劣势**：缺乏 mdnice 庞大的社区生态、插件系统（如复杂的数学公式、Mermaid 图表支持）以及账号云端同步功能，定位更偏向于轻量级的单机效率工具。
