### 🎯 一句话定位
一款基于 React 18 的纯前端 Markdown 编辑器，专为解决微信公众号排版痛点（如样式塌陷、图片防盗链、富文本粘贴脏数据）而生。

### ⚡ 核心功能
1. **富文本“魔法粘贴”**：底层基于 `turndown`，支持从飞书、Notion、Word 复制内容并自动“净化”为纯净 Markdown，支持图片直贴。
2. **微信底层兼容引擎**：内置 `wechatCompat` 逻辑，外链图片自动转 Base64 破解微信防盗链，重塑 Table/List 的 DOM 结构防止在微信后台塌陷。
3. **30套高定主题**：内置大量精调 CSS 主题（覆盖 Notion、少数派、Medium 等风格），一键切换，直接解决排版审美问题。
4. **多端预览与导出**：支持手机/平板/PC实时预览，支持导出为 PDF/HTML。

### 🎭 适用场景
1. **公众号运营与技术博主**：需要将带有代码块、复杂表格的技术文章，无损且美观地发布到微信公众号。
2. **跨平台内容分发**：主阵地在 Notion 或飞书，需要快速将富文本内容转换为 Markdown 并适配微信排版。
3. **轻量级文档/简历制作**：利用其丰富的内置主题和 HTML 转 PDF 功能，快速生成高颜值的文档。

### ✅ 是否值得深入研究

该项目是纯前端工具类开源的优秀范本。如果你正在研究 **Markdown 解析与渲染（`markdown-it`）**、**剪贴板事件劫持与 AST 转换（`turndown`）**，或是需要了解**微信公众号 DOM 兼容的黑魔法**（查看其 `src/lib` 目录下的核心逻辑），该项目源码非常干净，且配有完整的单元测试和 E2E 测试，极具参考价值。

### ⚠️ 主要缺点或风险
1. **Base64 图片体积陷阱**：将外链图片转 Base64 虽能绕过微信防盗链，但微信图文对单次粘贴的 HTML 体积有隐性限制。多图或高清大图场景下，极易导致粘贴失败或卡死。
2. **纯前端架构的数据脆弱性**：作为一个纯前端工具，若未深度集成 IndexedDB 等本地持久化方案，浏览器刷新或清理缓存极易导致未导出的稿件丢失。
3. **高度依赖微信规则**：核心价值之一建立在微信公众号后台现有的 DOM 解析漏洞/规则上。一旦微信升级其富文本编辑器，`wechatCompat` 的兼容逻辑可能瞬间失效。

### 🔗 与同类项目对比
* **对比 Markdown Nice (mdnice)**：mdnice 是该领域的老牌巨头，但目前商业化较重、UI 略显陈旧且代码库存在历史包袱。本项目采用现代技术栈（Vite + Tailwind），代码更轻量纯粹，视觉体验更现代。
* **对比 Doocs/md**：Doocs 偏向极客和基础排版，本项目在**“富文本逆向转换（魔法粘贴）”**和**“开箱即用的高定视觉主题”**上发力更深，更贴近现代内容创作者（如 Notion 重度用户）的痛点与审美。
