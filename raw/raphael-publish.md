### 🎯 一句话定位
一款专为微信公众号打造的现代 Markdown 排版引擎，通过“富文本转MD”和“DOM级微信兼容处理”，彻底解决公众号排版繁琐、代码高亮错乱及外部图片防盗链等痛点。

### ⚡ 核心功能（3~5条）
1. **“魔法粘贴”富文本净化**：底层基于 `turndown`，无缝对接飞书、Notion 等富文本源，粘贴瞬间自动清洗为纯净 Markdown，支持直接粘贴图片。
2. **一键适配公众号（核心壁垒）**：不仅是复制 HTML，其内置的 `wechatCompat` 引擎会将外链图片自动转 Base64 绕过微信防盗链（解决"图片来自第三方"报错），并重塑列表/表格 DOM 防止在微信中样式塌陷。
3. **30+ 高定排版主题**：内置从极简风（Mac/微信原生）到极客风（Dracula/Monokai）等 30 套独立设计的 CSS 样式，一键切换，告别同质化。
4. **多端预览与多图排版**：支持手机/平板/PC三端实时预览，支持多图网格布局，并可导出 PDF 和 HTML。

### 🎭 适用场景
1. **技术博主与自媒体**：习惯在 Notion、Obsidian 中用 Markdown 写作，需要快速分发到微信公众号且保持代码块和版面美观。
2. **公众号运营人员**：厌倦了 135、秀米等传统排版工具的臃肿，追求极简、高效的纯文本排版体验。
3. **前端开发者**：需要学习如何处理跨平台富文本剪贴板（Clipboard API）、Markdown 深度定制（Markdown-it），以及微信图文底层兼容的“黑魔法”。

### ✅ 是否值得深入研究
**值得（特别是对前端开发者和自媒体人）。**
**理由**：业务场景极度聚焦，直击痛点。代码库非常干净现代（React 18 + Vite + Tailwind），且工程化规范极佳（包含了 Vitest 单元测试和 Playwright E2E 测试）。其源码中的 `wechatCompat.ts`（微信兼容引擎）和 `htmlToMarkdown.ts`（剪贴板解析）是非常高价值的真实业务场景代码参考。

### ⚠️ 主要缺点或风险
1. **Base64 图片体积限制**：将外链图片转为 Base64 虽解决了防盗链，但如果文章包含大量高清大图，会导致剪贴板数据极其庞大，微信后台保存时可能卡顿或触发体积限制报错。
2. **排版上限受制于 Markdown**：无法实现类似“秀米”那种复杂的 SVG 交互动画或多层嵌套的异形布局，仅适合图文/代码类标准排版。
3. **微信规则变更风险**：高度依赖微信当前富文本编辑器的 DOM 解析规则，若微信未来升级编辑器底层黑名单过滤逻辑，可能导致部分样式失效，需持续跟进维护。

### 🔗 与同类项目对比
*   **对比 Markdown Nice (mdnice) / Doocs**：mdnice 是老牌王者，主题生态极其丰富，但目前商业化较重、代码相对陈旧。Raphael 更加轻量、现代化，且“魔法粘贴”体验更平滑，没有烦人的登录和广告限制。
*   **对比 135编辑器/秀米**：抛弃了拖拽式复杂 UI，完全基于 Markdown 语法，排版效率（尤其对代码、列表）呈指数级提升，但不适合做花哨的营销类视觉排版。
