### 🎯 一句话定位
一款开箱即用的纯前端 Markdown 编辑器，专为微信公众号排版设计，核心解决多源富文本转换、多主题渲染及微信图片防盗链/样式塌陷等痛点。

### ⚡ 核心功能
1. **富文本“魔法粘贴”**：底层基于 `turndown`，自动将飞书、Notion、Word 或网页的富文本（及截图）精准转化为纯净 Markdown，免除手动重写。
2. **微信公众号深度适配**：内置 `wechatCompat` 引擎，一键复制时自动将外链图片转为 Base64（突破微信防盗链），并重塑底层 DOM 解决列表/表格在微信中的塌陷问题。
3. **30+ 高定主题引擎**：内置模块化的 CSS 主题库（如 Notion风、少数派风、代码高亮等），一键切换，精准控制背景、排版及间距。
4. **多端预览与导出**：支持手机/平板/PC 实时响应式预览，提供所见即所得体验，并支持导出 PDF/HTML。

### 🎭 适用场景
1. **公众号创作者/运营**：习惯在 Notion、飞书或本地写稿，需要快速美化排版并无缝发布到微信公众号。
2. **技术博主**：需要带有优雅代码高亮和自定义样式的 Markdown 渲染工具，用于生成长图、PDF 或博客文章。
3. **前端开发者（源码学习）**：非常适合作为学习 React 18 + `markdown-it` 解析 + 富文本 AST 转换的优质参考项目。

### ✅ 是否值得深入研究
**结论：值得（特别是对前端工程师和内容创作者）。**

**理由**：
1. **痛点抓得极准**：直击微信公众号排版的恶心机制（图片防盗链、DOM 过滤），有实际业务价值。
2. **工程化极佳**：纯前端 SPA（无后端依赖），代码结构清晰，且罕见地包含了完整的单元测试（Vitest）和 E2E 测试（Playwright）。
3. **高价值源码**：其 `src/lib/wechatCompat.ts`（微信 DOM 兼容处理）和 `src/lib/htmlToMarkdown.ts`（富文本逆向解析）具有极高的直接复用价值。

### ⚠️ 主要缺点或风险
1. **Base64 图片的性能隐患**：依赖 Base64 解决图片防盗链，若文章包含大量高清大图，复制到微信后台时可能导致浏览器卡顿或触发微信的隐性体积限制。
2. **缺乏持久化存储**：作为一个纯前端工具，目前看来缺乏云端同步或可靠的本地历史版本管理，浏览器缓存清空可能导致未导出的稿件丢失。
3. **场景强绑定**：高度依赖微信公众号当前的富文本编辑器机制，若微信后台未来重构底层编辑器，其核心“一键复制”功能可能需要重写。

### 🔗 与同类项目对比
* **对比 mdnice (Markdown Nice) / Doocs/md**：
  * **优势**：相比 mdnice 日益臃肿和商业化，Raphael 更加轻量、现代（React 18 + Vite + Tailwind），UI 审美更符合现代直觉；它的“魔法粘贴”（富文本转 MD）体验比传统 MD 编辑器更平滑。
  * **劣势**：生态和社区插件不如 mdnice 丰富，缺乏多平台（知乎、掘金等）一键分发能力，功能纯粹聚焦于“排版”这一单点。
