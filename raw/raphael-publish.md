### 🎯 一句话定位
一款基于 React 的现代前端 Markdown 排版引擎，专门解决微信公众号排版痛点（防盗链、样式丢失），支持将任意富文本“无损”转为带主题样式的公众号图文。

### ⚡ 核心功能
- **富文本逆向解析（魔法粘贴）**：基于 `turndown` 深度定制，将飞书、Notion、网页等富文本一键转为纯净 Markdown，支持直接粘贴图片。
- **微信生态深度适配（wechatCompat）**：底层重塑 DOM 结构防止列表/表格在微信中塌陷；外链图片自动转 Base64，彻底绕过微信“此图片来自第三方”的防盗链限制。
- **开箱即用的高定主题**：内置 30 套精调 CSS 样式（如 Notion、GitHub、Linear、少数派风格），独立控制各元素表现，一键切换。
- **多端预览与导出**：支持 PC/Pad/Mobile 视口实时预览，支持基于 `html2pdf.js` 导出高清 PDF 或 HTML。

### 🎭 适用场景
1. **技术博主/独立开发者**：习惯使用 Markdown 写作，但受够了微信公众号后台糟糕的编辑器，需要快速将博客文章原样同步到公众号。
2. **新媒体内容运营**：在飞书/Notion 等协同工具中写稿，完成后直接复制粘贴，一键套用精美模板发布，统一团队视觉规范。
3. **极客简历/文档制作**：利用高质量的内置主题和 PDF 导出功能，快速制作排版精良的简历或技术报告。

### ✅ 是否值得深入研究
**值得（特别是前端工程师）。**

**理由**：
业务场景极度聚焦，直击痛点。工程化标准高（Vite + React 18 + TS，配备了完整的 Vitest 单元测试和 Playwright E2E 测试）。其源码中的 `wechatCompat.ts`（微信 DOM 兼容 hack 技巧）和 `htmlToMarkdown.ts`（富文本 AST 转换逻辑）对前端开发者处理富文本和跨平台 DOM 兼容有极高的参考价值。

### ⚠️ 主要缺点或风险
1. **Base64 图片的体积隐患**：将外链图片转为 Base64 虽然绕过了防盗链，但如果单篇文章图片过多或分辨率过大，可能导致复制到微信后台时浏览器卡顿，甚至触发微信单篇文章的体积/字符数限制。
2. **强依赖微信非公开规则**：底层的兼容引擎强依赖当前微信编辑器的 DOM 解析规则，若微信官方调整后台富文本过滤逻辑，可能导致排版瞬间失效，需持续跟进维护。
3. **纯前端无状态**：作为一个纯静态前端项目，无账户系统和云端存储，强依赖本地浏览器缓存，不适合跨设备接力编辑。

### 🔗 与同类项目对比
- **对比 mdnice (Markdown Nice)**：mdnice 目前商业化较重，许多高级主题和功能需登录或付费；Raphael 完全开源、纯前端计算、无广告，且 UI 更加现代化。
- **对比 doocs/md**：doocs/md 也是极其优秀的开源竞品，但 Raphael 的差异化优势在于**“魔法粘贴”（富文本逆向解析）**体验更顺滑，且其内置的 30 套主题在审美上更贴合现代互联网风格（如引入了 Linear、Notion、飞书等视觉规范）。
