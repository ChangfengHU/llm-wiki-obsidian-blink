### 🎯 一句话定位
一款专为微信公众号设计的现代化 Markdown 排版引擎，核心解决跨平台图文复制、样式定制及微信生态兼容（防盗链、DOM 塌陷）痛点。

### ⚡ 核心功能
1. **富文本“魔法粘贴”**：底层基于 `turndown`，支持将飞书、Notion、Word 等平台的富文本直接粘贴并净化为纯 Markdown，同时支持快捷键直贴图片。
2. **微信公众号深度兼容**：通过自研的 `wechatCompat` 引擎重塑底层 DOM，解决微信列表/表格塌陷问题；**外链图片自动转 Base64**，彻底绕过微信“图片来自第三方”的防盗链拦截。
3. **30套高定主题引擎**：内置多套精美的现代化样式（Notion、Medium、飞书等），通过独立的 CSS 作用域对排版元素进行精细化控制。
4. **多态导出与预览**：内置响应式设备预览（手机/平板/PC），并集成 `html2pdf.js` 支持一键导出高质量 PDF 和 HTML。

### 🎭 适用场景
- **微信公众号运营者**：在飞书、Notion 中写完初稿，需要一键无损排版并搬运至微信公众号后台。
- **技术博主/开发者**：习惯使用 Markdown 写作，需要优雅的代码高亮（`highlight.js` 支持）和极简的排版风格。
- **文档创作者**：需要将 Markdown 快速转化为排版精良的 PDF 报告进行跨平台分发。

### ✅ 是否值得深入研究
**值得（特别是前端工程师与工具链开发者）。**
- **业务价值**：痛点抓得极准，尤其是“图片 Base64 转换”和“微信 DOM 兼容”逻辑，直接击中自媒体人的刚需。
- **工程化优秀**：技术栈极新（React 18 + Vite 5 + Tailwind 3），代码结构清晰。不仅有单元测试（Vitest），还配置了 Playwright E2E 测试，是学习现代前端工程化和 AST 解析（Markdown 转换）的优秀脚手架项目。

### ⚠️ 主要缺点或风险
1. **Base64 转换的隐患**：虽然 Base64 解决了防盗链，但微信公众号后台对单篇文章的总字符数/体积有严格限制。如果文章包含大量高清长图，转 Base64 后极易导致保存失败或卡顿。
2. **高度依赖微信底层规则**：微信公众号后台的富文本编辑器如果更新 DOM 过滤规则或安全策略，项目中的 `wechatCompat.ts` 引擎可能随时失效，需要持续逆向维护。
3. **扩展性局限**：作为一个独立工具，目前缺乏类似 Obsidian 的插件机制，重度依赖作者内置的主题，用户自定义样式的自由度不如纯 CSS 注入工具。

### 🔗 与同类项目对比
相比于老牌的 **Markdown Nice (mdnice)** 或 **Doocs/md**：
- **差异化优势**：技术栈更现代，UI 设计审美更高级（紧跟 Notion、Linear 等现代设计语言），“魔法粘贴”的跨平台净化体验更顺滑。代码库极其干净，二次开发成本极低。
- **劣势**：生态和商业化沉淀不足。mdnice 拥有海量的 UGC 主题库和云端同步/分发能力，而本项目目前更偏向于一个纯粹、轻量的本地/静态工具。
