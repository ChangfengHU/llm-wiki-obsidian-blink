### 🎯 一句话定位
专为微信公众号创作者打造的纯前端 Markdown 排版引擎，提供“所见即所得”的多主题渲染，并彻底解决外部图片防盗链和微信排版格式错乱问题。

### ⚡ 核心功能
1. **魔法粘贴（富文本转 Markdown）**：复制飞书、Notion 或网页内容，粘贴时自动净化并转换为纯净 Markdown 代码，支持直接粘贴图片。
2. **微信生态特化适配**：一键复制到公众号后台，**自动将外链图片转 Base64 绕过防盗链**，并在底层重塑 DOM，解决列表/表格在微信中塌陷的问题。
3. **30套高定主题**：内置“纽约时报”、“飞书蓝”、“Notion”等 30 套精心设计的排版主题，一键切换，告别同质化。
4. **多图并排与多端预览**：支持安全的多图网格布局（微信内不折断），提供手机/平板/PC 三端实时预览。
5. **纯前端离线导出**：支持将排版结果直接导出为 PDF 或 HTML 文件。

### 🎭 适用场景
- **微信公众号运营者**：厌倦了微信自带编辑器或第三方排版工具的繁琐操作，希望用 Markdown 专注写作并一键生成高级排版。
- **跨平台内容创作者**：在 Notion、飞书或 Obsidian 中写完文章，需要无损搬运到微信公众号且保持排版美观一致。
- **开发者兼职博主**：习惯写代码风格的文章（包含大量代码块），需要工具完美高亮代码并适配微信后台。

### ✅ 是否值得深入研究
**值得**。
无论是作为实用工具还是前端源码学习，都非常有价值。对于前端开发者而言，该项目是学习**富文本与 Markdown 相互转换**（基于 `turndown` 和 `markdown-it`）、**AST 节点深度定制**，以及**特定平台（微信）DOM 兼容性黑魔法**（见 `src/lib/wechatCompat.ts`）的优秀开源范例。项目基于 React 18 + Vite 构建，代码结构清晰，且包含了完整的 Vitest 和 Playwright 测试，工程化水平极高。

### ⚠️ 主要缺点或风险
1. **Base64 图片的体积隐患**：为了绕过微信防盗链，将图片转为 Base64。如果文章包含大量高清大图，生成的 HTML 体积会暴增，可能导致剪贴板失效或微信后台卡顿。
2. **高度依赖微信编辑器规则**：其核心价值之一是解决微信排版塌陷。一旦微信官方更新富文本编辑器的底层过滤规则，该项目的 `wechatCompat` 引擎可能随时失效，需要被动跟进维护。
3. **纯前端架构的局限**：没有后端，意味着不支持云端草稿同步、用户自定义主题云端保存和图片图床上传管理。

### 🔗 与同类项目对比
- **对比 Markdown Nice (mdnice)**：mdnice 是老牌霸主，但目前商业化严重且代码较老。Raphael Publish 采用更现代的技术栈（React 18 + Tailwind），视觉设计更具有现代感（高定主题），且其“魔法粘贴”功能体验更优。
- **对比 Doocs/md**：Doocs/md 也是优秀的开源微信排版工具，但偏向基础化。Raphael Publish 的差异化优势在于**极佳的审美（主题设计）**和**更深度的微信兼容性处理**（如多图并列防折断、图片自动 Base64 处理）。
