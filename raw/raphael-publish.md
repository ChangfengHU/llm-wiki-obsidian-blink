### 🎯 一句话定位
一款专为微信公众号打造的纯前端 Markdown 排版工具，通过“黑科技”解决富文本转换、样式兼容和外部图片防盗链问题，实现“一次写作，完美粘贴到公众号”。

### ⚡ 核心功能
1. **富文本“魔法粘贴”**：底层基于 `turndown`，支持从飞书、Notion、网页直接复制富文本，粘贴瞬间自动净化为标准 Markdown，且支持直接粘贴截图。
2. **一键适配微信公众号**：内置 `wechatCompat` 引擎重塑 DOM，解决微信列表、表格排版塌陷问题；**外链图片自动转 Base64**，彻底绕过微信“图片来自第三方”的防盗链报错。
3. **开箱即用的高定主题**：内置 30 套精美排版样式（涵盖 Notion、少数派、NYT 风格等），包含对代码块、引用、表格的独立设计，一键切换。
4. **多端预览与导出**：支持手机/平板/PC实时响应式预览，并支持导出 HTML 和 PDF（基于 `html2pdf.js`）。

### 🎭 适用场景
- **微信公众号运营者**：厌倦了微信自带编辑器的难用，希望用 Markdown 高效写作并保持排版美观的创作者。
- **多平台内容分发**：在 Notion/飞书写完初稿，需要快速搬运、排版并发布到公众号的自媒体人。
- **技术博主**：需要频繁插入代码块，并要求代码在公众号中不变形、高精度高亮显示。

### ✅ 是否值得深入研究
**结论：值得（前端开发者与自媒体人必看）。**
- **对使用者**：这是一个极佳的、现代化的 mdnice 替代品，开箱即用，颜值极高。
- **对开发者**：代码规范极佳（React 18 + TS + Vitest + Playwright）。其源码中的 `src/lib/wechatCompat.ts`（微信 DOM 兼容处理）和 `src/lib/htmlToMarkdown.ts`（富文本转 MD 逻辑）对于需要处理**富文本编辑器、跨端复制粘贴兼容性**的前端工程师来说，具有极高的参考价值。

### ⚠️ 主要缺点或风险
1. **Base64 图片体积风险**：将外链图片转为 Base64 虽然解决了防盗链，但如果文章包含大量高清大图，生成的 HTML 体积会暴增，可能触发微信剪贴板或编辑器的字数/体积限制。
2. **依赖微信非公开规则**：针对微信的 DOM 兼容（如防塌陷）是基于当前微信编辑器的“Hack”，若微信后续大改底层编辑器，可能导致排版失效，需要持续维护。
3. **纯本地无云端**：作为一个纯静态 SPA，没有用户系统和云端存储，跨设备办公时无法同步历史草稿。

### 🔗 与同类项目对比
- **对比 Markdown Nice (mdnice)**：mdnice 商业化较重，UI 相对陈旧。Raphael 采用现代化的 Tailwind CSS 设计，颜值更高，且完全免费开源，没有烦人的广告或登录限制。
- **对比 Doocs/md**：Doocs/md 功能非常大而全。Raphael 则更聚焦于**“富文本向 Markdown 的逆向转换（魔法粘贴）”**以及**“高定主题”**，对非程序员出身的内容创作者更加友好，交互体验（如 Framer Motion 动画）更现代。
