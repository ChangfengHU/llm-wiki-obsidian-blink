### 🎯 一句话定位
一款专为微信公众号设计的现代 Markdown 排版引擎，通过“富文本自动转 MD”和“微信 DOM 兼容引擎”彻底解决公众号排版繁琐、图片防盗链和样式错乱的痛点。

### ⚡ 核心功能
- **富文本“魔法粘贴”**：底层基于 `turndown`，支持从飞书、Notion、Word 复制内容瞬间净化为纯净 Markdown，支持直接粘贴图片，免除手写语法。
- **微信原生级兼容 (`wechatCompat`)**：一键复制到公众号时，外链图片自动转 Base64 绕过防盗链限制；重塑底层 DOM，解决列表、表格在微信编辑器中塌陷的问题。
- **开箱即用的高定主题**：内置 30 套（如 Notion、少数派、硅谷风等）精调主题，元素级独立设计，告别同质化排版。
- **多端预览与多图排版**：支持手机/平板/PC实时预览，并提供适配微信不折断的多图网格布局引擎。

### 🎭 适用场景
- **技术博主/开发者**：习惯使用 Markdown 写作，需要快速将文章发布到微信公众号，且要求代码高亮完美、排版极客风。
- **新媒体运营**：需要将飞书、Notion 等协作工具中的文档快速搬运到公众号，免去手动重新排版的折磨。
- **内容归档与分发**：利用其纯净的渲染引擎，将文章一键导出为 PDF 或 HTML 进行存档或邮件发送。

### ✅ 是否值得深入研究
**值得（针对前端开发者/工具制作者）**。
理由：项目代码结构清晰，技术栈极度现代化（React 18 + Vite 5 + TS + Playwright）。其源码中的 `turndown` 深度定制逻辑（富文本转 MD）、`wechatCompat.ts`（微信 DOM 劫持与重塑）以及图片 Base64 处理策略，是开发 Web 生产力工具和富文本编辑器的绝佳参考。

### ⚠️ 主要缺点或风险
- **强依赖微信生态规则**：微信公众平台底层编辑器的任何大更新，都可能导致现有的 DOM 重塑和 Base64 注入策略失效（例如微信若限制 Base64 长度）。
- **纯前端架构无云端同步**：作为纯静态 SPA 工具，不具备云端草稿箱、多设备同步和图床管理功能，重度依赖本地浏览器状态。
- **自定义扩展性受限**：虽然内置 30 套主题，但缺乏可视化的 CSS 样式微调面板，高阶用户想要深度自定义样式需修改源码。

### 🔗 与同类项目对比
- **对比 mdnice / Doocs md 等老牌排版器**：
  - **优势**：UI/UX 更现代化、更丝滑；“魔法粘贴”（富文本反向解析）体验降维打击；外链图片自动转 Base64 实现了真正的零配置（小白无需折腾图床）。
  - **劣势**：缺少 mdnice 庞大的社区主题生态和插件系统；没有多平台一键分发能力，定位更偏向于“小而美的轻量级单机排版利器”。
