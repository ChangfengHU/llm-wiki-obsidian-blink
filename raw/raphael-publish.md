### 🎯 一句话定位
一款专为微信公众号设计的现代 Markdown 排版引擎，核心解决“富文本粘贴净化”与“微信后台DOM/图片防盗链兼容”两大痛点。

### ⚡ 核心功能
*   **魔法粘贴（反向解析）**：利用 `turndown` 将从飞书、Notion 等平台复制的富文本瞬间转化为干净的 Markdown，支持直接粘贴图片。
*   **微信原生兼容引擎 (`wechatCompat`)**：底层重塑 DOM 结构，解决微信编辑器中常见的列表和表格塌陷问题；**外链图片自动转 Base64**，彻底绕过微信“此图片来自第三方”的防盗链报错。
*   **30+ 高定主题**：内置基于 Tailwind 深度定制的多套排版样式（如 Notion、少数派、GitHub 风），覆盖多场景视觉需求。
*   **多端预览与导出**：支持手机/平板/PC实时预览，并基于 `html2pdf.js` 提供高质量的 PDF 和 HTML 导出。

### 🎭 适用场景
1.  **跨平台内容分发**：习惯在 Notion、Obsidian 或飞书中写作，需要无损、免排版直接发布到微信公众号的创作者。
2.  **极客/程序员写作**：厌倦了传统排版工具（如秀米、135编辑器）繁琐的拖拽操作，追求纯键盘、Markdown 极简体验的技术博主。
3.  **文档沉淀与分享**：需要快速将 Markdown 格式的技术文档、API 说明转换为精美 PDF 进行存档或邮件发送的团队。

### ✅ 是否值得深入研究
**值得（特别是针对前端开发者）。**
*   **业务参考价值极高**：如果你需要处理极其恶心的“微信公众号富文本兼容性”问题，其 `src/lib/wechatCompat.ts` 源码是现成的避坑指南。
*   **工程化规范**：项目虽然是个纯前端工具，但工程化极佳。采用了 Vite + React 18 现代技术栈，且配备了完整的单元测试（Vitest）和端到端测试（Playwright），是学习构建中小型开源工具的优秀模板。

### ⚠️ 主要缺点或风险
1.  **Base64 图片的隐患**：通过将图片转为 Base64 绕过防盗链虽然巧妙，但微信图文对 HTML 总字符数有严格限制（通常为 200W 字符）。如果文章包含大量高清大图，极易触发微信保存失败的报错。
2.  **黑盒依赖风险**：微信公众号的富文本过滤规则是动态且未公开的黑盒，项目的 `wechatCompat` 引擎未来可能随时因微信的暗改而失效，需要持续跟进维护。
3.  **纯前端无状态**：作为一个纯静态 SPA，缺乏云端同步和历史版本管理（除非完全依赖本地存储），不适合作为主力长文编辑器，更适合作为“发布前的最后一公里”工具。

### 🔗 与同类项目对比
*   **对比 Markdown Nice (mdnice)**：mdnice 是该领域的元老，但目前商业化较重、UI 相对陈旧。Raphael Publish 界面更现代（Framer Motion + Tailwind），且完全开源纯粹。
*   **对比 Doocs/md**：两者定位相似，但 Raphael Publish 在“富文本逆向解析为 Markdown（魔法粘贴）”这一功能上体验更顺滑，且主题设计更具现代审美（如引入了 Linear、Stripe 等硅谷风格）。
*   **对比 秀米/135 等传统工具**：Raphael 放弃了复杂的图文模块拖拽，专注于纯文本的优雅排版，受众更垂直（技术人员与文字创作者）。
