### 🎯 一句话定位
一款基于 React 的现代纯前端 Markdown 编辑器，专为微信公众号排版设计，核心解决富文本一键转 Markdown 及微信图文样式兼容痛点。

### ⚡ 核心功能
1. **魔法粘贴（富文本清洗）**：基于 `turndown`，直接从飞书、Notion、Word 复制内容粘贴，自动转化为纯净 Markdown，支持直接粘贴剪贴板图片。
2. **微信底层 DOM 适配**：内置 `wechatCompat` 引擎，自动将外链图片转为 Base64（彻底解决微信防盗链报错），并重塑列表/表格 DOM 防止在微信中塌陷。
3. **开箱即用的高定主题**：内置 30 套精心设计的 CSS 主题（如 Notion风、飞书蓝、少数派风），一键切换，告别原生白底黑字。
4. **多端预览与导出**：支持移动/平板/PC实时响应式预览，提供基于 `html2pdf.js` 的 PDF 和 HTML 导出功能。

### 🎭 适用场景
- **自媒体创作者/公众号运营**：习惯在 Notion 或飞书写稿，需要一键带样式无损发布到微信公众号。
- **技术博主**：需要将含有大量代码块（`highlight.js` 支持）的技术文章，优雅地粘贴到微信、知乎等富文本编辑器中。
- **前端开发者学习**：研究如何处理恶心的跨平台富文本剪贴板劫持、DOM 净化与转换。

### ✅ 是否值得深入研究
**值得（特别是前端工程师和富文本工具开发者）**

**理由**：
项目虽然是纯前端工具，但业务场景极度聚焦。其源码中的 `src/lib/wechatCompat.ts`（微信兼容处理）和 `src/lib/htmlToMarkdown.ts`（剪贴板富文本解析）具有极高的实战参考价值。项目工程化标准高（Vite + TS + Tailwind + E2E 测试），代码结构清晰，是学习 Markdown 解析（`markdown-it`）和富文本处理的优秀范例。

### ⚠️ 主要缺点或风险
1. **强依赖微信编辑器规则**：通过 Base64 绕过防盗链和 Inline CSS 注入是当前解法，若微信公众平台未来升级编辑器，严格限制 Base64 图片大小或过滤特定样式，核心功能将受影响。
2. **Base64 图片性能瓶颈**：长文若包含大量高清大图，全部转为 Base64 可能会导致复制时剪贴板卡顿，或触发微信后台的单篇文章字符数上限。
3. **缺乏持久化存储**：本质是一个“中转渲染器”，未见本地数据库或云端存储设计，不适合作为主力的文档管理工具。

### 🔗 与同类项目对比
- **对比 Markdown Nice (mdnice) / Doocs**：
  - **优势**：UI 更加现代化；“魔法粘贴”功能对其他笔记软件（飞书/Notion）的兼容性更好；**无需配置图床**，直接靠 Base64 解决图片防盗链，对小白更友好。
  - **劣势**：生态和知名度不如 mdnice，缺乏丰富的第三方社区主题和浏览器插件；不支持直接对接阿里云/腾讯云等专业图床。
