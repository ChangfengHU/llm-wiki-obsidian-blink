### 🎯 一句话定位
一款专为**微信公众号**打造的纯前端 Markdown 排版引擎，核心解决跨平台富文本粘贴、图片防盗链报错以及微信原生渲染塌陷的痛点。

### ⚡ 核心功能（3~5条）
1. **微信 DOM 级兼容（黑魔法）**：底层重塑列表、表格和多图网格的 DOM 结构，确保粘贴到微信后台不塌陷、不折断。
2. **图片防盗链破解**：一键复制时，自动将外链图片转为 Base64 注入剪贴板，彻底告别微信“此图片来自第三方”的红框报错。
3. **剪贴板逆向解析（魔法粘贴）**：基于 `turndown` 拦截剪贴板，从飞书、Notion、Word 复制的内容粘贴瞬间自动“净化”为纯净 Markdown。
4. **高定样式免写 CSS**：内置 30 套极具现代感的预设主题（如 Notion风、Claude风、少数派风），一键切换，所见即所得。

### 🎭 适用场景
- **技术/独立博主发文**：习惯在 Obsidian/Notion 写 Markdown，需要无损、带高级代码高亮地分发到微信公众号。
- **运营团队内容搬运**：将外部来源（飞书文档、网页、Word）的杂乱富文本，一键清洗并格式化为标准公众号排版。
- **多端静态归档**：利用其纯净的渲染引擎，将 Markdown 快速导出为带样式的 PDF 或 HTML 进行存档或邮件发送。

### ✅ 是否值得深入研究
**值得（特别是前端开发者）。**
- **理由**：该项目代码结构极度清晰，且配备了完整的单元测试（Vitest）和 E2E 测试（Playwright），工程素养极佳。它的 `src/lib` 目录是学习**剪贴板事件劫持**、**Markdown AST 解析 (`markdown-it`)** 以及**针对特定平台（微信）做 DOM 降级适配**的绝佳源码级教科书。

### ⚠️ 主要缺点或风险
1. **Base64 转换的隐患**：将所有外链图片转为 Base64 复制，如果文章包含大量高清大图，可能导致剪贴板体积过大，低配电脑复制时会卡顿，甚至触发微信编辑器的字符上限。
2. **极度依赖微信编辑器规则**：其核心价值建立在微信现有的富文本解析规则上（`wechatCompat.ts`）。一旦微信公众平台更新底层编辑器（如收紧标签白名单），项目可能瞬间失效，需要持续跟进维护。
3. **纯本地无云端**：作为纯静态 SPA 站点，缺乏云端同步和多设备草稿漫游能力，更偏向于“排版工具”而非“创作中心”。

### 🔗 与同类项目对比
- **对比 Markdown Nice (mdnice)**：mdnice 是老牌霸主，生态庞大但日益商业化（登录墙、广告）。Raphael Publish 胜在**极致轻量、完全开源无广告、UI 更符合现代审美**（如引入了 Tailwind 和 Framer Motion），且在“富文本转 Markdown”的体验上更平滑。
- **对比 Doocs/md**：Doocs 偏向传统技术风，而本项目在**视觉品味**（30套现代主题）和**工程现代化**（React 18 + Vite + TS 严格模式）上表现更优秀。
