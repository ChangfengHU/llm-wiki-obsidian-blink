### 🎯 一句话定位
一款专为微信公众号设计的现代 Markdown 排版工具，通过“富文本自动逆向转 Markdown”和“微信样式兼容引擎”，彻底解决跨平台内容发布的排版痛点。

### ⚡ 核心功能（3~5条）
1. **富文本净化（魔法粘贴）**：基于 `turndown` 深度定制，支持从飞书、Notion、Word 直接复制内容或图片，粘贴即自动转换为纯净 Markdown，免去手动重排。
2. **微信公众号一键适配**：底层包含专门的 `wechatCompat` 引擎，外链图片自动转 Base64 绕过微信防盗链，重塑 DOM 解决列表和表格在微信中塌陷的问题。
3. **30套高定主题**：内置基于 Tailwind CSS 打造的高质量主题（如 Notion风、Medium风、代码暗黑风等），一键切换，告别公众号同质化排版。
4. **多端预览与导出**：支持手机/平板/PC实时响应式预览，并集成 `html2pdf.js` 支持导出为 PDF，兼顾网页发布与文档存档。

### 🎭 适用场景
- **跨平台创作者**：习惯在 Notion、飞书、Obsidian 中写作，需要无损、美观地同步发布到微信公众号的作者。
- **技术博主/开发者**：需要展示优雅代码高亮（highlight.js），且对排版审美有较高要求（如硅谷风、极客风）的程序员。
- **轻量级文档转换**：需要快速将 Markdown 或富文本片段转换为精美的 PDF 报告或 HTML 单页的场景。

### ✅ 是否值得深入研究
**结论：值得（特别是前端工程师）。**

**理由**：
该项目是一个极佳的**纯前端工具型产品**源码范例。它展示了如何处理复杂的富文本与 Markdown 互转、如何编写针对特定平台（微信富文本编辑器）的 DOM 兼容“黑魔法”，以及如何利用 React 18 + Vite 构建流畅的本地 SPA。项目包含完整的单元测试（Vitest）和 E2E 测试（Playwright），工程化规范极高，代码极具参考价值。

### ⚠️ 主要缺点或风险
1. **大图 Base64 转换瓶颈**：通过 Base64 绕过微信防盗链是把双刃剑。如果文章包含大量高清大图，生成的 Base64 字符串会极其庞大，可能导致剪贴板崩溃、浏览器卡顿，或触碰微信编辑器的底层字符限制。
2. **高度依赖微信后台规则**：微信公众号编辑器的底层规则（如允许的 CSS 属性、DOM 嵌套限制）若发生暗改，该工具的 `wechatCompat` 引擎会瞬间失效，需要极高的维护跟进频率。
3. **纯无状态工具**：无云端存储和账号系统，仅适合作为发布前的“排版中转站”，不能作为主力知识管理工具。

### 🔗 与同类项目对比
- **对比 Markdown Nice (mdnice)**：mdnice 是老牌霸主，生态繁荣，但目前商业化较重，UI 相对传统。Raphael Publish 胜在 UI 极度现代化、交互更丝滑（魔法粘贴），且完全开源无广告。
- **对比 Doocs/md**：Doocs/md 同样是优秀的开源微信排版器，但 Raphael Publish 在“富文本逆向解析（飞书/Notion粘贴兼容）”和“开箱即用的高定主题”上做得更极致，更偏向现代内容创作者的审美。
