### 🎯 一句话定位
一款专为微信公众号设计的现代 Markdown 排版引擎，通过“富文本一键转纯净 Markdown”和“突破微信图片防盗链”的核心机制，彻底解决创作者跨平台（Notion/飞书 -> 微信）的排版痛点。

### ⚡ 核心功能
- **“魔法粘贴”富文本净化**：基于 `turndown` 引擎劫持剪贴板，支持从飞书、Notion、Word 复制内容并自动转为纯净 Markdown，支持直接粘贴截图。
- **微信底层 DOM 适配**：一键复制到公众号时，**自动将外链图片转为 Base64**，彻底解决微信“此图片来自第三方”的报错；同时重塑列表和表格 DOM，防止在微信内塌陷。
- **高定主题与多图网格**：内置 30 套高审美 CSS 主题（如 Notion、硅谷风、少数派），并支持微信兼容的多图并排网格布局。
- **纯前端多端预览与导出**：支持手机/平板/PC 实时预览，并基于 `html2pdf.js` 提供高质量的 PDF 和 HTML 导出。

### 🎭 适用场景
1. **公众号创作者**：习惯在 Notion、飞书、Obsidian 中写作，需要一键无损发布到微信公众号，且不想手动传图。
2. **技术博主**：需要发布包含大量代码块、自定义高亮样式的技术文章，并要求格式在微信内不乱码。
3. **轻量级文档输出**：需要快速将 Markdown 内容套用精美模板，转换为 PDF 分享给客户或团队。

### ✅ 是否值得深入研究
**值得。**
- **作为工具**：开箱即用，审美极佳，直击微信排版的痛点，可以直接作为日常生产力工具。
- **作为源码**：技术栈非常干净（React 18 + Vite + TS + Tailwind）。其源码中的 `src/lib/htmlToMarkdown.ts`（剪贴板劫持与富文本逆向解析）和 `src/lib/wechatCompat.ts`（微信 DOM 兼容黑魔法）极具参考价值，是前端学习**富文本处理**和**跨端剪贴板/DOM 适配**的优秀范例。

### ⚠️ 主要缺点或风险
1. **Base64 图片体积限制**：将外链图片转为 Base64 虽然绕过了防盗链，但如果文章包含大量高清大图，可能导致生成的 HTML 体积过大，触发微信公众号后台的字数/字符数上限。
2. **纯静态无云端**：纯前端项目，无账号体系和云同步功能。它是一个完美的“排版中转站”，但不能替代主力笔记软件。
3. **强依赖微信规则**：底层 DOM 注入高度依赖当前微信公众号后台的解析规则，若微信大幅更新编辑器，项目可能需要被动跟进维护。

### 🔗 与同类项目对比
- **对比 Markdown Nice (mdnice)**：mdnice 商业化较重（有广告、付费主题）；本项目完全开源免费，UI 更现代，且“富文本转 Markdown（接管非Markdown平台的复制）”的能力更强。
- **对比 Doocs/md**：Doocs 也是优秀的开源微信排版竞品。相比之下，本项目的差异化优势在于**现代化的视觉品味**（30套主题设计感更强）以及对 **Notion/飞书生态的无缝粘贴支持**。
